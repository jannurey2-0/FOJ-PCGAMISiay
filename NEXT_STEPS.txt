ROADMAP & NEXT STEPS TO MAKE FOJ-PCGAMI SIAY FULLY FUNCTIONAL
============================================================

Moving this high-fidelity prototype into a production-ready, fully functional 
church platform requires the following architectural additions:

1. AUTHENTICATION & SECURITY (COMPLETED & SECURED)
--------------------------------------------------
* [COMPLETED] Secured triggers on user signup: Default role to 'Church Member' and 'approved = false' to prevent role spoofing.
* [COMPLETED] Prevented privilege escalation: Added trigger check (`check_profile_escalation`) in Supabase to restrict self-escalation of roles or approval status.
* [COMPLETED] Removed insecure bootstrap emails (e.g. admin@pcgami.org). First admin must be manually updated in the database.
* [NEXT STEP] Apply similar database Row Level Security (RLS) checks on new tables (e.g., chat_messages, rsvps, gratitude_notes) as they are created.

2. BACKEND & PERSISTENT DATABASE
---------------------------------
* Provision a relational database (like PostgreSQL via Supabase) or NoSQL database (like Firestore) to persist:
  - User profile details & roles.
  - Thanksgiving RSVP lists and headcount.
  - Chat messages and user reaction counts.
  - Photo records and Gratitude Wall entries.
* Replace the local React arrays in `App.tsx` with fetch calls/mutations (such as TanStack Query / Axios).

3. REAL-TIME COMMUNITY CHAT
----------------------------
* Replace the simulated local messaging with a real-time transport layer.
* Use WebSockets (e.g., Socket.io, Firebase Realtime Database, or Supabase Realtime Channels).
* Implement push notifications for critical ministry mentions (e.g., tagging @media-team or @worship-team).

4. CLOUD MEDIA STORAGE
-----------------------
* The current photo uploader uses object URLs (`URL.createObjectURL`), which only exist in local memory.
* Set up a cloud object storage bucket (e.g., AWS S3, Cloudinary, or Supabase Storage).
* Implement file verification (mime-types, file size caps) before uploading to protect the cloud infrastructure.

5. CONTENT MODERATION PIPELINE
-------------------------------
* Ensure Gratitude Wall notes and Photo submissions are stored with an `approved: false` status by default.
* Implement a server-side web-socket notification or dashboard badge for admins when new moderation tasks arrive.

6. DEPLOYMENT & CI/CD
---------------------
* Deploy the frontend build folder (`/dist`) to hosting providers like Vercel, Netlify, or AWS Amplify.
* Connect a custom domain name (e.g., `foj-pcgami-siay.org`).
